# Route Output

Write confirmed insights to their destinations: auto-memory and/or vault.

## Input

- `confirmed_insights`: JSON array of insights the user approved
- `project_name`: Project name for auto-memory file naming (e.g., "kinso")
- `vault_path`: Path to vault (default `{{VAULT}}/`)
- `memory_path`: Path to auto-memory (derived from `CLAUDE_PROJECT_DIR`)

## Process

### Project-State Insights → Auto-Memory

1. Check if `{memory_path}/project_{project_name}.md` exists.
2. **If exists:** Read the current file. Merge new project-state insights into the existing content:
   - Update sections that cover the same topic (e.g., "Current load" dates/counts).
   - Append genuinely new sections.
   - Preserve manually-added context (anything not generated by ingest).
   - Update the `description` frontmatter field to reflect current state.
3. **If doesn't exist:** Create the file with standard frontmatter:
   ```yaml
   ---
   name: project_{project_name}
   description: {one-line summary}
   type: project
   ---
   ```
4. Update `MEMORY.md` index if a new file was created.

### Durable Insights → Vault Notes

For each `durable-insight`:

1. Search for existing vault notes on the same topic using:
   ```bash
   node {{PLUGIN}}/scripts/vault-search.mjs search "<key terms>" --rerank
   ```
2. If a closely matching note exists, skip (don't duplicate — mention in summary).
3. If novel, spawn a `note-writer` subagent (subagent_type: `learning-loop:note-writer`) with:
   - **insight**: The title
   - **research**: The body + source context
   - **destination**: `0-inbox/`
   - **related_notes**: Top vault search results as wiki-links

### Project Index Update

If `{vault_path}/4-projects/{project_name}.md` exists:
- Read it and update the "Active TODOs" section if present.
- Update "Recent Decisions" if any durable insights are decisions.
- Don't restructure. Don't add sections that don't exist.

## Output

Return a summary:

```
Ingest complete:
  Memory: updated project_{name}.md (N items)
  Vault: wrote N notes to 0-inbox/ (M skipped as duplicates)
  Index: updated 4-projects/{name}.md
```

## Rules

- Use `Write` tool for all file I/O. Never use Obsidian MCP tools.
- Never overwrite the entire auto-memory file. Read first, merge, write.
- Don't create a project index note if one doesn't exist — that's a manual decision.
- Vault notes go through the `note-writer` agent to ensure voice consistency.
